Text Editor

The text editor I chose is VIM. I chose VIM because I have seen many programmers use it and have heard great things about it. I worked through the tutorial included with VIM (vimtutor) because it was 1) easy to access and 2) extremely interactive.

The tutorial covered how to navigate VIM using the 'hjkl' keys, the G operation to jump to specific line numbers, and using the 'we$0' keys to navigate along a line. It also covered the commands used for deleting, replacing, copying, and pasting text; these are all 'Normal Mode' commands. Along with these it covered writing (saving) the file, inserting contents of another file into the current one, and also running shell commands from within the VIM. Overall, the tutorial showed me that VIM is a very powerful text editor and I plan to use it for the majority of my career as a computer scientist.

Version Control System

The VCS I chose is git. I chose git because it is the most popular VCS on the market and I have used it before, although not extensively. I watched this tutorial from Lynda.com in its entirety to learn about using git locally and then watched a bit of Chapter 13 from this tutorial, also from Lynda.com, to learn about using github repositories.

The first tutorial covered how to setup a local git repository, track and stage files, and make commits. It also briefly covered branching and how to merge branches. Chapter 13 of the second tutorial discussed creating and using a github repository. This is an online repository that can be accessed by anyone (or any machine) by 'pulling' those files onto a local machine.
